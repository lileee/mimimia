# 魔法少女「月辉灵猫召唤」网页原型设计说明

- 日期：2026-07-13
- 状态：设计已由用户逐段确认
- 目标平台：桌面网页，鼠标操作
- 交付目标：完整项目文件与可公开分享的网页链接

## 1. 项目目标

制作一个以视觉表现为优先、可由访问者亲手操作的 2.5D 网页原型。玩家按住鼠标约 2.5 秒完成蓄力，脚下魔法阵逐层展开；蓄满后松开，召唤出「月辉灵猫」。

角色使用原创分层 2D 美术，魔法阵、粒子、光线、薄雾、镜头与召唤过程均实时生成。原型应在高性能设备上提供完整画质，并在设备能力不足时自动降低效果，但保留全部互动流程。

## 2. 明确不包含的内容

- 不制作完整游戏、剧情关卡或战斗系统。
- 第一版只包含一名魔法少女、一个魔法阵和一种召唤法术。
- 不制作完整 Live2D、逐帧动画或 3D 角色模型。
- 不以手机端为主要目标。
- 不加入用户账号、存档、排行或后台服务。
- 不直接使用用户提供的参考图作为最终素材。

## 3. 已确认的方向

| 项目 | 决定 |
| --- | --- |
| 核心体验 | 玩家按住鼠标蓄力，松开施法 |
| 角色表现 | 分层 2D 角色与实时 3D 魔法效果组合 |
| 法术结果 | 召唤小型灵猫使魔 |
| 视觉主题 | 月辉灵猫：紫银、月相、星轨、安静神秘 |
| 角色原型 | 白发小魔女、大帽、斗篷裙、法杖；全部细节原创重构 |
| 角色比例 | 5.5 头身 |
| 角色气质 | 优雅、自信、仪式感强 |
| 魔法阵位置 | 脚下主阵 |
| 镜头 | 轻微俯视、完整全身 |
| 场景 | 月光虚境 |
| 召唤方式 | 灵猫暗影升起，再由星光从下向上填满 |
| 蓄力规则 | 约 2.5 秒蓄满；提前松开失败并柔和消散 |
| 成功状态 | 灵猫留在少女身边，漂浮并看向鼠标 |
| 重玩方式 | 玩家主动点击「再次施法」 |
| 声音 | 环境音乐与完整施法音效；无少女语音 |
| 画质取向 | 视觉效果优先，普通设备约 30 帧可接受 |
| 回退方式 | 自动进入简化画质，保留完整召唤流程 |
| 技术原则 | 最终效果优先，Three.js 为主体，TSL 用于适合的实时效果 |

## 4. 玩家体验流程

### 4.1 启动与进入

1. 页面显示月相加载动画与真实加载进度。
2. 角色、灵猫、阵纹和关键声音准备完成后，显示「进入月光虚境」。
3. 玩家点击按钮后启动音乐并进入等待状态。该点击同时满足浏览器对声音播放的用户操作要求。
4. 第一次进入只显示一句提示：「按住鼠标，凝聚月光。」

### 4.2 等待状态

- 少女进行缓慢呼吸和自然眨眼。
- 头发、帽饰、斗篷、丝带与裙摆分别轻微摆动。
- 月光虚境中只有稀疏星尘、薄雾和远处柔和月轮。
- 魔法阵尚未显现，音乐保持安静循环。

### 4.3 蓄力阶段

- **0～0.8 秒**：阵心亮起，第一圈月相符号被逐笔描绘。
- **0.8～1.7 秒**：外圈展开并反向旋转，星尘向少女聚拢。
- **1.7～2.5 秒**：全部阵纹点亮，衣物受能量影响扬起，镜头缓慢靠近。
- **蓄满后继续按住**：保持完成状态并给出明确声音和光线提示；不设置过度蓄力或反噬。

角色、魔法阵、粒子、镜头和声音均读取同一蓄力进度，保证同步。

### 4.4 提前松开

- 不召唤灵猫。
- 魔法阵从外向内柔和熄灭，播放轻微消散音。
- 约一秒后恢复等待状态，可以立即重试。
- 不显示惩罚、扣分或负面提示。

### 4.5 成功召唤

1. 玩家蓄满后松开。
2. 画面出现极短停顿以强调释放瞬间。
3. 阵心升起灵猫暗色轮廓。
4. 月光和星尘从脚部向上填满轮廓。
5. 灵猫形成实体并漂浮到少女肩膀附近。
6. 灵猫的耳朵、尾巴轻动，视线轻微跟随鼠标。
7. 页面只保留「再次施法」与声音按钮，等待玩家主动重置。

## 5. 美术设计

### 5.1 魔法少女

- 5.5 头身，完整全身展示。
- 主色：象牙白、深夜蓝、月光紫；淡金色仅用于收边和法杖。
- 白发与浅蓝眼睛，表情平静、自信。
- 大帽内侧为星空纹理，外侧使用月相装饰。
- 斗篷裙保留童话感，但服装结构、花纹和配饰全部重新设计。
- 参考图中的十字与宗教图案全部替换为原创月相、猫眼和星轨符号。
- 法杖顶部采用圆环包裹弯月，中心嵌入猫眼形月石。
- 站姿优雅稳定，不采用夸张战斗姿态。

角色至少拆分为：后发、身体与腿、脸部、双眼、前发、帽身、帽饰、斗篷外层、星空内衬、左右丝带、裙摆、法杖及法杖光源。各层保持统一画风、统一光向和透明边缘。

### 5.2 月辉灵猫

- 小型、圆润、可爱，漂浮时不遮挡角色脸部。
- 深紫与银白为主色，浅蓝眼睛与少女呼应。
- 额头带原创月相印记。
- 尾巴末端形成弯月轮廓。
- 耳尖和脚部带轻微星尘渐变。
- 分离头部、眼睛、耳朵、身体、尾巴和发光层，以支持轻量动画。

### 5.3 魔法阵

魔法阵不使用一张静态完整图片，实时组合为：

1. 阵心：原创猫眼月石图案。
2. 中圈：月相变化符号。
3. 外圈：星轨、原创文字和细小刻度。
4. 辅助层：径向刻度、微小符文与星点。

三层主要圆环采用不同方向与速度旋转。蓄力进度控制其绘制范围、亮度、透明度和旋转速度。

### 5.4 月光虚境

- 深紫黑背景，远处柔和月轮，少量薄雾与星尘。
- 镜头轻微俯视，角色居中并保留完整全身。
- 特效主要位于脚下、角色轮廓外侧和身后，避免遮挡脸部与服装。
- 高亮集中在阵心、法杖与灵猫显形位置，保持视觉层级。

### 5.5 素材来源

- Image 2 用于原创角色、灵猫、分层透明素材和少量纹样参考。
- Three.js/TSL 用于魔法阵、光线、粒子、薄雾、扭曲与召唤过程。
- 所有音乐与音效必须为原创或具有明确可用授权，并记录来源。

## 6. 技术路线

### 6.1 版本与基础工具

- 直接使用 Three.js，不引入额外 3D 框架层。
- 新项目以 TypeScript 与 Vite 组织本地开发、构建和静态发布。
- 设计基线固定为 Three.js r185。实现期间不自动漂移版本；升级前必须重新运行高画质与回退测试。
- 使用 `WebGPURenderer`：支持时优先 WebGPU，不支持时由同一渲染器使用 WebGL 2 后端。
- 使用 TSL 编写核心跨后端材质与动画逻辑。
- 使用 `RenderPipeline` 组织辉光、轻微色差、画面柔化与最终输出。
- 不使用 `ShaderMaterial`、`RawShaderMaterial`、`onBeforeCompile()` 或旧 `EffectComposer` 作为核心路径。

### 6.2 运行层次

**高画质模式**：

- 完整粒子数量与高分辨率输出。
- 完整辉光、空间扭曲、残影与细致薄雾。
- 可使用 WebGPU 专属能力增强粒子，但不得承担关键召唤步骤。

**兼容画质模式**：

- 减少粒子和薄雾层数。
- 降低内部渲染精度与辉光强度。
- 关闭最昂贵的空间扭曲和可选 WebGPU 专属增强。
- 保留完整角色、阵纹、失败消散、成功召唤和灵猫互动。

开发期间必须提供强制 WebGL 2 回退测试入口，保证兼容路径不是理论存在。

## 7. 模块边界

### 7.1 舞台模块

负责场景、镜头、月轮、背景星尘和全局光线。对外接收画质配置和镜头进度，不管理施法状态。

### 7.2 角色模块

负责加载角色分层素材并执行呼吸、眨眼、摆动和蓄力响应。对外接收时间、蓄力进度与召唤阶段。

### 7.3 魔法阵模块

负责创建阵心、中圈、外圈和符文，接收蓄力进度并更新绘制范围、透明度、亮度与旋转。

### 7.4 粒子模块

负责背景星尘、聚拢星尘、失败消散和成功爆发。使用对象池或实例化方式复用粒子，避免每次施法重新创建大量对象。

### 7.5 召唤模块

只管理「暗影升起—星光填充—灵猫就位」的演出顺序，以及召唤完成后的耳朵、尾巴和视线反应。

### 7.6 声音模块

独立管理环境音乐、蓄力音层、失败音和成功音。关闭声音不改变动画时间轴。

### 7.7 画质模块

判断设备能力，选择初始画质；运行期间只在持续卡顿时逐级降级，并尽量在一次演出结束后应用变化。

### 7.8 页面界面模块

负责加载进度、进入按钮、首次操作提示、声音按钮、兼容画质标识、错误提示和「再次施法」。界面与实时画面分离。

## 8. 状态与数据流

状态依次为：

`loading → entry → idle → charging → charged → summoning → complete → resetting → idle`

提前松开走独立分支：

`charging → dissolving → idle`

规则：

- 只有 `idle` 可以开始蓄力。
- `charging` 使用单一蓄力进度 `0..1` 驱动所有表现。
- 到达 `1` 后进入 `charged`，继续按住不会变化为失败。
- `charged` 松开才进入 `summoning`。
- `summoning` 期间忽略新的施法输入。
- `complete` 只接受鼠标位置与重置操作。
- 页面失去焦点或收到取消操作时，`charging`/`charged` 进入 `dissolving`，不会误触发召唤。

## 9. 声音设计

- 环境音乐：安静、空灵、月夜感，进入页面后循环。
- 0～0.8 秒：低频月光底音。
- 0.8～1.7 秒：加入细小晶体与符文旋转声。
- 1.7～2.5 秒：逐步增加高频星尘与能量上升声。
- 蓄满：清晰但不刺耳的完成提示。
- 提前松开：柔和下降与消散声。
- 成功：短暂静音空间后出现清亮月光提示音，再接灵猫成形声。
- 不加入少女念咒语音。

## 10. 加载、异常与恢复

### 10.1 加载

- 显示真实进度，不用假进度条。
- 关键资源完成后才开放进入按钮。
- 在第一次施法前预先准备核心材质和效果，降低首演卡顿。
- 初始资源压缩后尽量控制在约 15 MB 内。

### 10.2 资源错误

- 音乐失败：以静音状态继续。
- 次要装饰失败：跳过，不阻塞体验。
- 角色或灵猫失败：自动重试一次，仍失败则显示「重新加载」。
- 不允许以黑屏作为错误结果。

### 10.3 操作中断

- 鼠标移出、指针取消、切换标签页或窗口失焦时，当前蓄力安全消散。
- 返回页面后恢复等待状态，不补触发召唤。

### 10.4 图形环境异常

- 尝试重建场景和资源。
- 无法恢复时显示「重新进入」按钮与简短说明。

### 10.5 运行中卡顿

- 持续卡顿后逐级减少粒子、画面精度和光晕强度。
- 避免在召唤演出中途产生明显跳变，优先在演出结束后应用降级。

## 11. 测试与验收

### 11.1 功能测试

- 等待状态只能通过按住主鼠标键开始蓄力。
- 2.5 秒前松开必定消散，不生成灵猫。
- 蓄满后保持按住不会失败。
- 蓄满松开必定完整召唤。
- 召唤期间不能重复触发。
- 成功后灵猫保持存在并响应鼠标。
- 「再次施法」完整清理上次效果并回到等待状态。
- 静音切换不影响动画时间。

### 11.2 视觉测试

- 重点检查 1920×1080 和 1440×900 桌面视口。
- 角色脸部与主要服装细节不被阵纹和粒子遮挡。
- 高画质与兼容画质保持相同构图、颜色方向和关键时间点。
- 对等待、蓄力中段、蓄满、失败、显形中段和完成状态保留截图基准。

### 11.3 性能测试

- 目标测试电脑在 1920×1080 高画质下保持约 30 帧以上，无长时间冻结。
- 强制 WebGL 2 回退时，完整流程可操作并保持约 30 帧目标。
- 多次重复施法后，内存和对象数量不持续增长。
- 首次施法不出现明显材质编译卡顿。

### 11.4 异常测试

- 模拟音乐失败、次要素材失败和关键素材失败。
- 在不同蓄力时刻测试鼠标移出、标签页切换和窗口失焦。
- 测试图形环境恢复与无法恢复的提示流程。
- 在最新桌面浏览器上验证高画质或自动回退结果。

## 12. 制作阶段

1. 原创角色与灵猫概念稿、定稿和分层素材。
2. 月光虚境、镜头与交互状态骨架。
3. 正式角色图层动画和灵猫待机动画。
4. 魔法阵、蓄力粒子、失败消散和召唤显形。
5. 声音、镜头、颜色和后期细节统一。
6. 兼容画质、异常恢复、完整测试和公开发布。

每一阶段必须在进入下一阶段前具有可运行或可查看的验收结果。

## 13. 主要风险与控制方式

| 风险 | 控制方式 |
| --- | --- |
| 新版渲染器仍在快速发展 | 固定 r185；升级前运行完整高画质与回退测试 |
| WebGPU 专属能力无法回退 | 核心召唤只使用两端验证通过的 TSL；专属能力仅作增强 |
| 第一次施法卡顿 | 启动阶段预先准备材质与效果 |
| 分层角色边缘不一致 | 概念定稿后统一生成、统一光向，并逐层合成检查 |
| 透明图层与粒子互相遮挡 | 固定层级与深度规则，保存关键画面截图基准 |
| 粒子过多导致填充压力 | 分级粒子数量、动态画质、内部精度调整；上采样只作后备方案 |
| 两种画质结果差异过大 | 使用同一状态、同一构图和同一时间轴，只减少非关键效果 |
| 音频授权不清晰 | 只使用原创或明确授权素材并保存记录 |
