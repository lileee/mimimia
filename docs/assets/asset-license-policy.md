# 素材公开与登记规则

## 适用范围

所有进入 `art/source/` 或 `public/assets/` 的非代码文件，都必须在 `asset-ledger.csv` 中恰好登记一次。运行时素材只有在 `public_use_status` 为 `approved` 时才能进入 `public/assets/`。

## 原创图像

- Image 2 生成图必须记录生成日期、对应提示文件、生成方式和人工修改。
- 少女与灵猫先各自确认一张完整母图，再从同一母图拆层；不得分别生成互不一致的部件。
- 人工补画、透明边缘整理、颜色统一和拆层均写入修改记录。
- 私有参考图只保留在被排除的本地目录，不进入公开台账、运行时资源、源码包、截图或发布包。

## 音乐与音效

- 只允许原创制作或具有清晰公开使用依据的声音。
- 每条声音记录作者、创作或生成方法、后期处理和授权依据。
- 禁止使用来源不明的影视、游戏或网络采样，也不加入少女语音。

## 字体、纹理与图标

- 页面优先使用系统字体，避免额外字体授权和传输成本。
- 纹理与图标优先程序生成或原创制作。
- 如使用第三方素材，台账必须写明作者、来源、许可证和原文链接；依据不清晰时阻止构建与发布。

## 校验值与发布

- 台账中的 `sha256` 必须与实际文件一致。
- 大幅 `.ora` 源文件和无损 `.wav` 源文件使用 Git LFS；运行时 WebP/MP3 直接进入版本记录，保证静态页面可读取。
- 每次新增、替换或修改素材后，都要重新运行私有边界和台账检查。
