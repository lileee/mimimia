# 开发说明

## 项目如何运行

页面只有一条主流程：加载原创素材并预热画面，等待用户进入，按住鼠标蓄力，提前松开则消散，蓄满后松开则显形灵猫，最后等待再次施法。

所有画面模块读取同一份“当前帧信号”，其中包含时间、状态、蓄力、消散、显形和鼠标位置。这样法阵、粒子、人物、灵猫、声音和后期不会各自计时而逐渐错位。

## 主要边界

| 部分 | 责任 | 主要位置 |
| --- | --- | --- |
| 入口与编排 | 加载、预热、进入、画质、恢复和页面生命周期 | `src/main.ts`、`src/app/` |
| 状态 | 约束等待、蓄力、蓄满、消散、召唤、完成和重置 | `src/state/` |
| 输入 | 只接受鼠标左键，处理移出、取消、失焦和标签页隐藏 | `src/input/` |
| 场景 | 镜头、背景、层级和逐帧更新 | `src/stage/` |
| 角色 | 分层少女、眨眼和轻微待机动作 | `src/character/` |
| 法术 | 法阵、粒子池、灵猫显形与视线 | `src/effects/`、`src/summon/` |
| 渲染 | WebGPU 优先、WebGL 2 回退和三档后期 | `src/rendering/` |
| 声音 | 用户手势解锁、环境音乐、循环层、提示音和静音 | `src/audio/` |
| 画质 | 三秒初始判断、运行监测、延后降级和冷却 | `src/quality/` |
| 恢复 | 图形丢失后的重建与可见失败结果 | `src/recovery/` |

## 三档画质

固定配置在 `src/quality/qualityProfiles.ts`。初始稳定帧率达到 45 选择高画质，30 至 44.9 选择均衡画质，低于 30 选择兼容画质。运行中要连续两个 5 秒窗口低于 27 才请求降低一档；演出进行时延后，回到安全状态再切换，两次切换至少相隔 30 秒。

## 开发入口

```bash
npm ci
npm run dev
```

开发版支持以下查询参数，正式构建默认关闭：

- `?debug=1`：显示后端、档位、有效帧率、状态和对象统计；
- `?quality=high|balanced|compatibility`：固定画质；
- `?backend=webgl2`：强制兼容后端；
- `?experienceState=...`：开发截图使用的固定状态；
- `?fault=...`：自动测试使用的资源或图形失败入口。

## 修改后的最小检查

| 改动 | 至少运行 |
| --- | --- |
| 状态、输入、画质或声音 | 对应单元测试与完整流程测试 |
| 渲染、法阵、粒子或召唤 | 单元测试、强制 WebGL 2 完整流程、视觉检查 |
| 图层或声音素材 | 素材台账、私有边界、图层或声音验证 |
| 发布相关 | 构建、自动验收、体积检查、发布包验证 |

完整发布回归命令见 [发布说明](DEPLOYMENT.md)。
